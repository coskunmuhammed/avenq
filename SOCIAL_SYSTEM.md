# AVENQ Social Content & Distribution System v2.0

Standard operating procedures and content templates for LinkedIn, X (Twitter), GitHub, Instagram, and YouTube. All content must maintain AVENQ's signature unhyped, engineering-first authority.

---

## 1. Distribution Channels

### LinkedIn (Professional & Institutional)
- **Focus**: Engineering decision logs, product philosophy, system architecture specs.
- **Tone**: Restrained, objective, structural.
- **Format**: Text-first posts, PDF slide decks (monochrome dark background, monospaced typography).

### X / Twitter (Engineering Notes & Updates)
- **Focus**: Concise architectural assertions, benchmark logs, code snippets, release changelogs.
- **Tone**: Sharp, direct, zero filler.

### GitHub (Open Standards & Documentation)
- **Focus**: Repositories, public schemas, API specs, deployment guides.
- **Tone**: Technical, rigorous, thoroughly documented.

---

## 2. Post Templates

### Template 1: Engineering Note / Decision Log
```
// ENGINEERING NOTE: [Topic]

[1-2 sentences stating the problem]

Our architectural decision:
- [Fact 1]
- [Fact 2]
- [Fact 3]

Software should solve operational problems permanently.

avenq.pro/docs
```

### Template 2: Product Release / Update
```
// PLATFORM RELEASE: [Product Name] v[Version]

[1 sentence core definition]

System updates deployed:
1. [Update 1]
2. [Update 2]
3. [Update 3]

Full release notes: avenq.pro/docs#changelog
```

---

## 3. Visual Specifications for Posts
- **Background**: `#0B0B0B` or `#141414`.
- **Border**: 1px `rgba(255, 255, 255, 0.12)` outline.
- **Typography**: Inter / System Sans paired with SFMono / Monospace headers.
- **Color Accents**: Pure `#FFFFFF` text, `#10B981` status active, `#FBBF24` in-development. Zero gradients.
